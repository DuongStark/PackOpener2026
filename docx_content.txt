1. Tổng Quan Sản Phẩm
1.1 Mục Tiêu Sản Phẩm
Xây dựng backend cho web game mở thẻ cầu thủ, cho phép người chơi đăng ký, đăng nhập, mua pack, mở pack, quản lý inventory, bán thẻ, và theo dõi lịch sử giao dịch. Admin quản lý toàn bộ dữ liệu hệ thống qua module Admin API.
1.2 Vòng Lặp Kinh Doanh
Mua Pack → Mở Pack → Nhận Thẻ → Bán Thẻ lấy Coin → Tiếp tục Mua Pack

2.4 Bảng: pack_card_pool
Danh sách thẻ có thể ra trong mỗi pack cùng trọng số để weighted random.
📌 Quy tắc Weighted Random: weight là số nguyên dương, đơn vị tùy ý. Xác suất ra 1 thẻ = weight_thẻ / tổng_weight_toàn_pool. Ví dụ: pool có 3 thẻ weight [60, 30, 10] → xác suất lần lượt 60%, 30%, 10%. Tổng KHÔNG bắt buộc phải = 100. Server phải tổng hợp và tính runtime, không hardcode. Nếu pool rỗng → throw lỗi 500 và log cảnh báo.

3. Business Rules
Mã
Mô Tả
Module
BR-01
Email đăng ký phải là duy nhất trong hệ thống. Kiểm tra case-insensitive.
Auth
BR-02
Mật khẩu tối thiểu 6 ký tự. Validate bằng class-validator trước khi hash.
Auth
BR-03
Đăng nhập thành công trả về JWT access token có hạn 24 giờ (86400 giây).
Auth
BR-04
Endpoint cần xác thực phải yêu cầu JWT hợp lệ. Trả 401 nếu thiếu hoặc sai token.
Auth
BR-05
Chỉ pack có is_active = true mới được hiển thị trong GET /packs và cho phép mua.
Pack
BR-06
Mua pack: trừ coin users.balance và tạo user_packs trong cùng một Prisma transaction. Nếu fail rollback toàn bộ.
Pack
BR-07
Mở pack phải idempotent: nếu user_packs.status đã là 'OPENED', trả về kết quả cũ từ pack_opening_results, KHÔNG random lại.
Pack
BR-08
Không được random lại kết quả cho cùng một userPack đã mở. Kiểm tra status TRƯỚC khi chạy random logic.
Pack
BR-09
Random card dựa trên weighted random từ pack_card_pool. Mỗi lần random 1 thẻ, lặp card_count lần. CÓ THỂ ra trùng thẻ trong 1 lần mở nếu pool cho phép.
Pack
BR-10
Bán thẻ: set status = 'SOLD' trên N inventory_items (IN_INVENTORY) + giảm inventory.quantity (xóa row nếu về 0) + cộng users.balance + ghi transactions trong cùng một Prisma transaction.
Sell
BR-11
Lịch sử giao dịch (transactions) và kết quả mở pack (pack_opening_results) là dữ liệu bất biến — không UPDATE, không DELETE.
Tất cả
BR-12
Inventory phân trang với limit mặc định 20, tối đa 100. Trả 400 nếu limit > 100.
Inventory
BR-13
Không được bán thẻ không thuộc sở hữu của user hiện tại. Kiểm tra inventory record tồn tại với user_id = currentUserId.
Sell
BR-14
Không được bán số lượng vượt quá số thẻ đang có. Kiểm tra inventory.quantity >= requested quantity trước khi trừ.
Sell
BR-15
Pack opening result phải lưu snapshot card_snapshot (JSONB) tại thời điểm mở, bao gồm toàn bộ stats và sellPrice.
Pack
BR-16
User mới được cấp 1000 coin khởi tạo khi đăng ký thành công. Tạo 1 transaction loại INITIAL_CREDIT cùng lúc.
Auth
BR-17
Mọi thao tác thay đổi users.balance phải được bọc trong Prisma transaction. Không update balance bên ngoài transaction.
Tất cả
BR-18
Client không được truyền tham số random vào quá trình mở pack. Server hoàn toàn kiểm soát logic random.
Pack
BR-19
Khi bán hết 1 loại thẻ (quantity về 0): xóa row inventory thay vì để quantity = 0. Đảm bảo GET /inventory/:cardId trả 404 cho thẻ đã bán hết.
Sell
BR-20
GET /inventory/summary và GET /inventory/:cardId phải đăng ký route đúng thứ tự trong NestJS: summary trước, :cardId sau.
Inventory
BR-ADM-01
Tất cả endpoint /admin/* phải kiểm tra JWT hợp lệ VÀ role = 'ADMIN'. Dùng RolesGuard + @Roles('ADMIN') decorator trong NestJS.
Admin
BR-ADM-02
Admin không thể tự xóa, tự khóa, hoặc tự hạ quyền tài khoản của chính mình. Server kiểm tra userId trong JWT so với target userId.
Admin
BR-ADM-03
Khi admin điều chỉnh balance trực tiếp qua PATCH /admin/users/:id, phải ghi 1 transaction loại ADMIN_ADJUSTMENT vào bảng transactions. amount = (newBalance - oldBalance), description = 'Admin balance adjustment'.
Admin
BR-ADM-04
Không cho phép xóa pack đã có user_packs. Admin phải dùng isActive = false để ẩn pack.
Admin
BR-ADM-05
Không cho phép xóa thẻ đang có trong pack_card_pool hoặc inventory của user. Trả 409 với số liệu cụ thể.
Admin
BR-ADM-06
Không cho phép xóa pool entry cuối cùng của 1 pack (pack sẽ không thể mở). Pool phải có ít nhất 1 thẻ.
Admin
BR-ADM-07
Không cho sửa card_count của pack nếu đã có ít nhất 1 user_pack liên quan (dù là PENDING hay OPENED).
Admin
BR-ADM-08
GET /admin/stats/dashboard được cache 5 phút (Redis hoặc in-memory). Không query trực tiếp DB mỗi request.
Admin
BR-ADM-09
Khi admin khóa tài khoản (isActive = false), user đang online sẽ bị ngắt kết nối WebSocket ngay lập tức (server emit event 'account_suspended' trước khi disconnect).
Admin
BR-ADM-10
Enum ADMIN_ADJUSTMENT phải được thêm vào cột type của bảng transactions (bổ sung vào BR-16). Data model transactions.type cập nhật: 'BUY_PACK' | 'SELL_CARD' | 'INITIAL_CREDIT' | 'ADMIN_ADJUSTMENT'.
Admin + Data Model
4. API Contract Chi Tiết
🔒 JWT: truyền qua header Authorization: Bearer <token>  |  🌐 Public: không cần auth  |  🔐 Admin: yêu cầu JWT + role=ADMIN  |  Tất cả request/response body là JSON (Content-Type: application/json)
📌 Convention lỗi thống nhất: { "statusCode": number, "message": string | string[], "timestamp": "ISO8601", "path": "/endpoint" }

6. WebSocket Contract
⚙️ Stack: Socket.IO qua @nestjs/websockets + @nestjs/platform-socket.io | Namespace: /game | Auth: JWT qua handshake auth header | Ping interval: 25s | Ping timeout: 60s
6.1 Xác Thực Kết Nối
Client gửi JWT khi khởi tạo kết nối. Server xác thực trong handleConnection. Từ chối ngay nếu thiếu hoặc sai token. Nếu tài khoản bị khóa (isActive = false), server disconnect ngay với code ACCOUNT_SUSPENDED.
Client code: const socket = io("wss://api.example.com/game", { auth: { token: "Bearer eyJhbGci..." }, transports: ["websocket", "polling"] });

6.5 Business Rules WebSocket
Mã
Mô tả
BR-WS-01
Server chỉ push event vào room riêng room-{userId}, KHÔNG broadcast toàn server.
BR-WS-02
Idempotency WS: nếu client emit pack:open_request cho userPack đã OPENED → server push lại kết quả cũ (không random lại) + emit pack:open_completed ngay.
BR-WS-03
Mỗi client chỉ được mở tối đa 1 pack cùng lúc. Nếu có pack đang xử lý → server từ chối yêu cầu mới, trả error RATE_LIMITED.
BR-WS-04
Delay giữa các pack:card_revealed PHẢI được cấu hình server-side (không để client kiểm soát). Giá trị mặc định: 500ms/thẻ.
BR-WS-05
balance:updated phải được push đồng thời với REST response — không trễ hơn 200ms so với thời điểm transaction hoàn thành.
BR-WS-06
Khi client disconnect giữa chừng: server vẫn tiếp tục xử lý và lưu kết quả vào DB. Client reconnect → dùng REST GET /user-packs/:id hoặc /history/:id để lấy lại kết quả.
BR-WS-07
Rate limit: mỗi user tối đa 10 WebSocket emit/giây. Vượt quá → nhận event error với code RATE_LIMITED.
BR-WS-08
Khi admin khóa tài khoản user đang online: server emit account:suspended (với reason và timestamp) trước, sau đó gọi client.disconnect(). Đây là event mới thêm từ v2.3.

7.1 Bảo Mật
Mã
Yêu cầu
Triển khai gợi ý
SEC-01
Password phải được hash bằng thuật toán an toàn
bcrypt với cost >= 12, hoặc argon2id
SEC-02
Không trả về thông tin nhạy cảm trong response

Validate input ở tất cả endpoint

SEC-04
Rate-limit auth endpoint chống brute-force

Idempotency cho open pack chống replay

Admin API phải kiểm tra role trong mọi request
Dùng RolesGuard + @Roles('ADMIN') decorator. Không dựa vào client truyền role.
